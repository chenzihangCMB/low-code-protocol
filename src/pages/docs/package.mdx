---
title: 软件包
description: 软件包是低代码平台产出的最小单位，由元数据、微代码、微服务组成。
---

本协议约定软件包的目录结构、版本号、软件包依赖等规则；约定软件包的发布、安装与升级规范；约定软件包仓库的规范。

## 软件包的定义

- 软件包名称
- 版本号
- 描述
- 依赖的软件包
- 依赖的外部软件包

## 软件包内容

- 元数据
  - 对象
  - 字段
  - 列表视图
  - 简档
  - 权限集
  - 页面布局
  - 报表
  - 流程
- 微代码
  - 按钮
  - 表单事件
  - 触发器
- 微服务
- 微页面

## 源码版本管理

软件包以源码仓库的形式存在，通过 Gitlab、Github 等工具实现版本管理。

## 元数据的开发

元数据使用可视化工具设计，通过VS Code同步工具同步为代码。

## 微代码的开发与调试

微代码使用代码编辑器（例如VS Code）编写。微代码 deploy 到 metadata service 后执行。

- 触发器在 Record Service 中执行。
- 前端微代码在浏览器中执行。

> 微代码的运行环境需支持单步追踪与调试功能。

## 微服务的开发与调试

微服务独立部署与运行，可以使用任何语言开发。

## 打包、发布

软件包测试通过后，发布到软件包仓库。
- Java 可用 Maven 仓库
- NodeJS 可用 NPM 仓库
